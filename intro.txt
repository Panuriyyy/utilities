принудительное создание связи в бд

-- 1. Временно отключаем проверку существующих данных
ALTER TABLE [dbo].[Soderganie_zakaza] NOCHECK CONSTRAINT ALL;

-- 2. Создаём внешний ключ с NOCHECK (без проверки старых данных)
ALTER TABLE [dbo].[Soderganie_zakaza] 
WITH NOCHECK 
ADD CONSTRAINT [FK_Soderganie_zakaza_Tovarr] 
FOREIGN KEY ([id_tovara]) REFERENCES [dbo].[Tovarr] ([id_tovara]);

-- 3. Включаем проверку обратно (новые данные будут проверяться)
ALTER TABLE [dbo].[Soderganie_zakaza] CHECK CONSTRAINT [FK_Soderganie_zakaza_Tovarr];

узнать в бд -- Имя сервера и базы данных одной строкой
SELECT @@SERVERNAME AS ServerName, DB_NAME() AS DatabaseName;

--- A) Проверить, установлен ли .NET 8 SDK ---

Откройте PowerShell или CMD и выполните:

  dotnet --version

Должно показать 8.x.x (например 8.0.100).
Если команда не найдена или версия ниже 8 — установите SDK:

  winget install Microsoft.DotNet.SDK.8

После установки закройте и снова откройте терминал, снова проверьте:

  dotnet --version


--- B) Перейти в папку проекта ---

  cd "ПУТЬ\К\ПРОЕКТУ\savegad"

Пример:

  cd "e:\Новая папка\спаси  и сохрани\ЗАДАНИЕ ДЭ\готово\еще раз\savegad"


--- C) Установить ВСЕ библиотеки проекта одной командой (РЕКОМЕНДУЕТСЯ) ---

В папке проекта уже есть файл savegad.csproj со списком пакетов.
Достаточно выполнить:

  dotnet restore

Эта команда скачает все нужные NuGet-пакеты:
  - Microsoft.EntityFrameworkCore.SqlServer  (работа с SQL Server)
  - Microsoft.EntityFrameworkCore.Design     (миграции, дизайн БД)
  - Microsoft.EntityFrameworkCore.Tools      (инструменты EF)
  - Microsoft.AspNetCore.Session             (сессии, авторизация)
  - System.Drawing.Common                    (обработка фото товаров)

Затем проверьте сборку:

  dotnet build

Если «Сборка успешно завершена» и «Ошибок: 0» — библиотеки установлены правильно.


--- D) Запуск приложения ---

  dotnet run

Или через скрипт (если порт занят — освободит автоматически):

  powershell -NoProfile -ExecutionPolicy Bypass -File .vscode\start.ps1

Сайт откроется: http://localhost:5191


--- E) Если dotnet restore не помог — установить пакеты вручную ---

Выполните по очереди в папке проекта:

  dotnet add package Microsoft.EntityFrameworkCore.SqlServer --version 8.0.0
  dotnet add package Microsoft.EntityFrameworkCore.Design --version 8.0.0
  dotnet add package Microsoft.EntityFrameworkCore.Tools --version 8.0.0
  dotnet add package Microsoft.AspNetCore.Session --version 2.2.0
  dotnet add package System.Drawing.Common --version 8.0.0

Потом снова:

  dotnet restore
  dotnet build


--- F) SQL Server (база данных) ---

Приложение подключается к SQL Server. На ПК должен быть установлен
SQL Server или SQL Server Express.

Проверка (если установлен sqlcmd):

  sqlcmd -S ИМЯ_СЕРВЕРА -E -Q "SELECT 1"

Имя сервера укажите в appsettings.json (сейчас: Server=Honurij).

Установка SQL Server Express (если нет):

  winget install Microsoft.SQLServer.2019.Express


--- G) Частые ошибки в терминале ---

  Ошибка: dotnet не является командой
  → Установите .NET 8 SDK (см. пункт A)

  Ошибка: address already in use (порт 5191 занят)
  → powershell -File .vscode\start.ps1
  или:
  Get-NetTCPConnection -LocalPort 5191 | ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }

  Ошибка при dotnet build: файл заблокирован
  → Закройте запущенное приложение (Ctrl+C в терминале) и снова dotnet build

  Ошибка подключения к БД
  → Проверьте SQL Server запущен и строка в appsettings.json верная
